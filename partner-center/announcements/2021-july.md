## <a name="new-commerce-experience-in-csp-for-azure-expanding-to-government-community-cloud-gcc"></a><a name="16"></a>Azure için CSP'de yeni ticari deneyim Government Community Cloud (GCC)

### <a name="summary"></a>Özet

24 Ağustos 2021'de, Azure için Bulut Çözümü Sağlayıcısı (CSP) programı kapsamındaki yeni ticaret deneyimi tüm Government Community Cloud (GCC) iş ortakları tarafından kullanılabilir olacak.

### <a name="categories"></a>Kategoriler

- Tarih: 2021-07-26
- Özellikler

### <a name="impacted-audience"></a>Etkilene hedef kitle

GCC CSP programı aracılığıyla işlemden geçen iş ortakları

### <a name="details"></a>Ayrıntılar

## <a name="enhancement-of-tool-for-transfer-of-azure-subscriptions-between-partners-in-the-cloud-solution-provider-csp-program"></a><a name="15"></a>Bulut Çözümü Sağlayıcısı (CSP) programında iş ortakları arasında Azure aboneliklerinin aktarımı için araç geliştirmesi

### <a name="summary"></a>Özet

Mevcut Azure aboneliği geçiş aracı, müşterinin CSP abonelikleri içinde önceki Azure teklifinin iş ortakları arasında aktarılamayacak şekilde geliştireceğiz.

### <a name="categories"></a>Kategoriler

- Tarih: 2021-07-21
- Özellikler

### <a name="impacted-audience"></a>Etkilene hedef kitle

CSP doğrudan fatura iş ortakları ve dolaylı sağlayıcıları ve dolaylı sağlayıcıları aracılığıyla kurumsal bayiler

### <a name="details"></a>Ayrıntılar

21 Temmuz 2021'den başlayarak, mevcut [Azure](./2020-september.md#15) aboneliği geçiş aracı müşterinin CSP abonelikleri içinde önceki Azure teklifinin iş ortakları arasında aktarılamalarını sağlarken bu abonelikleri yeni Azure teklifi planına da aktaracak şekilde geliştireceğiz. İş ortaklarının almaları gereken araç ve süreç, iş ortağının iş ortağına yeni Azure teklifi planı aktarımlarıyla aynı kalacaktır. İş ortağı kılavuzunda ayrıntılı [bilgileri bulabilirsiniz.](https://partner.microsoft.com/resources/detail/step-by-step-guide-transfer-azure-subscriptions-between-csp-partner-new-commerce-experience-pdf)

Azure aboneliği geçiş araçları, CSP programında Azure için yeni ticaret deneyimini genişleterek CSP iş ortaklarının Azure'ın müşterilerin gelişen iş ihtiyaçlarına uyum sağlanmış bir şekilde yönetmesini kolaylaştırır. İş ortakları, Azure geçişleri için kullanımı kolay bu self servis aracıyla zaman tasarrufu sağlar. Faturalama sahipliğinin geçişi sırasında Azure hizmetlerinde kesinti yaşanmaz.

#### <a name="key-capabilities-and-considerations"></a>Önemli özellikler ve dikkat edilmesi gerekenler

- Bir aboneliği önceki Azure teklifinden yeni Azure teklifi planına değiştirmek faturalama para birimini değiştirebilir. Yeni Azure teklifinde iş ortakları, Avrupa [Birliği/Avrupa](./2021-june.md#9)Serbest Ticaret Birliği ve Birleşik Krallık bölgeleri dışında müşterilerin konum para biriminde faturalandıracak ve ağustos ayından itibaren bu bölgelerdeki tüm iş ortakları, iş ortaklarının konum para birimi cinsinden faturalandırı.

- Azure rezervasyonları aktarnmaz.

- Geçiş, müşteri kararına göre gerçekleşir ve yalnızca her iki iş ortağı da açıkça eyleme geçerek gerçekleşir.

- Müşterinin kapalı kalma süresi veya Azure hizmetleri üzerinde herhangi bir etkisi olmaz ve geçiş hemen uzer.

- Bu araç deneyimi İş Ortağı Merkezi doğrudan fatura iş ortakları ve dolaylı sağlayıcılar tarafından kullanılabilir. Kurumsal bayilerin bu aracı kendi adına kullanmak için dolaylı sağlayıcılarla birlikte çalışması gerekir.

### <a name="next-steps"></a>Sonraki adımlar

## <a name="changes-to-azure-offer-availability-related-to-the-new-commerce-experience-in-csp-for-azure"></a><a name="14"></a>Azure için CSP'de yeni ticaret deneyimiyle ilgili Azure teklifi kullanılabilirliği değişiklikleri

### <a name="summary"></a>Özet

Bu iletişim, yeni ticaret deneyimi için Azure teklif kullanılabilirliği Bulut Çözümü Sağlayıcısı (CSP) yapılan bir güncelleştirmedir.

### <a name="categories"></a>Kategoriler

- Tarih: 2021-07-21
- Özellikler

### <a name="impacted-audience"></a>Etkilene hedef kitle

CSP programı aracılığıyla işlem yapılan iş ortakları

### <a name="details"></a>Ayrıntılar

CSP programında önceki Azure teklifi kullanılabilirliği değişiklikleri.

Ocak ayında iş ortaklarına CSP programında önceki Azure teklifi kullanılabilirliğiyle ilgili yapılacak değişiklikler hakkında bilgi verdik. Ocak duyuru [ayrıntıları için ayrıca Bkz. Ocak 2021](./2021-january.md#12) duyuruları.

Önceki Azure teklifinde üç değişiklik aşaması olacağını duyurduk ve bu üç aşamanın her biri başlangıç tarihine altı ay öncesinden iş ortaklarını bilgilendireceğiz.

Ocak ayında duyurulmuştu gibi, 1. aşamanın başlangıç tarihi bugün, 21 Temmuz 2021'tir. Ayrıca 2. aşamanın başlangıç tarihi olan 1 Şubat 2022'nin de olacağını da açıklarız.

#### <a name="three-phases"></a>Üç aşama

**1. Aşama: 21 Temmuz 2021'den itibaren**

1. aşamada, tüm yeni Azure CSP programı işleri yeni Azure teklifinde (Azure planı) olacak. Aşağıdaki yönergeler geçerlidir:

- CSP programında iş ortakları ve müşteriler arasındaki tüm mevcut kurumsal bayi ilişkileri için, müşteri önceki Azure teklifini zaten satın aldı ise iş ortağı işlem yapmaya devam eder ve müşteri bu iş ortağından önceki Azure teklifini satın almaya devam eder.

- Önceki Azure teklifi artık CSP programında işlem yapılan yeni iş ortakları tarafından kullanılamaz.

- Önceki Azure teklifi artık CSP programında bir iş ortağı aracılığıyla satın alan yeni müşteriler tarafından kullanılamaz.

- Tüm yeni kurumsal bayi ilişkileri veya yeni müşteri alımları için iş ortakları yalnızca yeni Azure teklifini (Azure planı) işlemden kullanabilir.

- İş ortakları ve müşteriler arasındaki tüm mevcut kurumsal bayi ilişkileri için, müşteri daha önce Azure'a hiç satın almadı ise, iş ortağı yalnızca işlemde bulunarak yeni Azure teklifini yalnızca satın alır.

**2. Aşama: 1 Şubat 2022'den itibaren**

2. aşamada teşvikler ve iş ortağı marjı fırsatı önceki Azure teklifinden kaldırılacaktır.

CSP'de önceki Azure teklifi

- CSP'de önceki Azure teklifiyle ilişkili iş ortağı marjı fırsatı ve teşvikleri kaldırılacak.

- CSP'de önceki Azure teklifinin fiyatı, Azure için tahmini perakende fiyatına eşit olur.  

CSP'de yeni Azure teklifi (Azure planı)

- İş ortakları CSP'de yeni Azure teklifi (Azure planı) üzerinde iş ortağı tarafından kazanılan kredi (PEC) kazanmaya uygun olmaya devam edecektir.  

- Microsoft Commerce Incentives iş ortakları CSP'de yeni Azure teklifi (Azure planı) ile ilgili teşvikler için uygun olmaya devam edecektir.  

- Gümüş bulut platformu yetkinliklerine sahip iş ortakları, CSP'de (Ekim 2021'den itibaren geçerli) yeni Azure teklifi (Azure planı) ile ilgili ek teşvikler için uygun olacak.

**3. Aşama: Duyuru tarihi**

3. aşamada, önceki Azure teklifinde kalan tüm müşteriler CSP programında (Azure planı) yeni Azure teklifine geçirilir. Önceki aşamalarda olduğu gibi, 3. aşama başlangıç tarihini iş ortaklarına altı ay önceden ileteceğiz.

#### <a name="partner-earned-credit"></a>İş ortağı kazanılmış kredisi

Yeni ticaret deneyimiyle iş ortakları, PEC modeli aracılığıyla sürekli olarak farklı katma değerli hizmetler için tanınır. PEC rewards iş ortakları, sürdürülebilir kârlılık oluşturan katma değerli hizmetler aracılığıyla müşteri başarısının yollarına odaklanmaya odaklanır.

## <a name="introducing-windows-365-hybrid-windows-for-a-hybrid-world"></a><a name="13"></a>365 Windows ye tanıtma: Hibrit Windows karma dünya için hibrit bulut

### <a name="summary"></a>Özet

Microsoft, Windows [365](https://aka.ms/M365EndpointPartner)ile Windows getiriyor.

### <a name="categories"></a>Kategoriler

- Tarih: 2021-07-20
- Modern Çalışma Alanı

### <a name="impacted-audience"></a>Etkilene hedef kitle

Bulut Çözümü Sağlayıcısı (CSP) programı aracılığıyla işlem yapılan iş ortakları

### <a name="details"></a>Ayrıntılar

14 Temmuz'da Microsoft, [Windows 365](https://aka.ms/M365EndpointPartner)ile buluta Windows duyurdu. Windows 365, Windows 10 veya Windows 11'de deneyim yaşamanın, kişiselleştirilmiş uygulamalarınızı, ayarlarınızı ve içeriğinizi Microsoft bulutundan herhangi bir cihaza akışa alamanıza yönelik yeni bir yol sunar.

Windows 365 Cloud PC'nize hoş geldiniz:

- **Cihazlar arasında akış**

    Bulut bilgisayarınıza anında önyükler ve cihaz Windows akışı yapabilirsiniz.  
- **Her zaman hazır ve güncelleştirilmiş**

    Buluttan akışı yapılan ve birlikte seyahat eden Windows 365, nerede olursanız olun, çalışmak istediğiniz cihazda her zaman hazırdır ve her zaman güncelleştirilir. Bu, Microsoft bulutu tarafından Windows ve çok yönlü bir deneyimdir.
- **Kişisel ve tanıdık**

    Windows 365 kişiselleştirilmiş ve tüm ayarlarınız ve içeriğiniz hakkında bilgi sahibidir. Bu nedenle, Bulut bilgisayarınıza geri dönüp onu bıraktığınız yerdedir.
- **Ölçeklenebilir ve esnek**

    Windows 365, bulutun gücü ve bulut bilişim ilkelerini kullanarak tasarım Sıfır Güven. Bilgiler güvenli hale getirildi ve cihazda değil bulutta depolanır.
- **Tahmin edilebilir maliyetler**

    Windows 365, Bulut bilgisayarları satın almayı kullanıcı başına aylık fiyatlandırmayla tahmin edilebilir hale gelir.

- **Satın alma, dağıtma ve yönetme basit**

    Windows 365, diğer çözümlerden farklı olarak arkalarında sanallaştırma kullandığı için tedarik, dağıtım, yönetim ve güncelleştirmeleri basitleştiriyor.

Windows 365, müşterilerin 2 Ağustos 2021'den başlayarak [windows365.com, Kurumsal Anlaşma](https://windows365.com)veya Bulut Çözümü Sağlayıcısı satın almaları için kullanılabilir.

### <a name="next-steps"></a>Sonraki adımlar

- Microsoft Inspire [Windows 365 duyurularını](https://aka.ms/MSInspireBRK101) izleyin.
- blog gönderimiz aracılığıyla Windows 365 hakkında [daha fazla bilgi edinebilirsiniz.](https://www.microsoft.com/microsoft-365/blog/?p=237041?)
- 365'i kullanarak [](https://aka.ms/Windows365PartnerWebinar) iş ortağı fırsatı hakkında daha fazla bilgi edinmek için 18 Ağustos'ta İş ortağı pazara Windows katılın.

Bulut Çözümü Sağlayıcısı programı aracılığıyla işlem Bulut Çözümü Sağlayıcısı iş ortakları  

### <a name="details"></a>Ayrıntılar

Ekim 2021'den başlayarak Microsoft, api'leri çağıran iş ortakları için bir zaman aralığı içinde daha tutarlı bir performans için API İş Ortağı Merkezi uygulayacak.Kaynakların fazla kullanımına engel olmak için azaltma, bir hizmete yönelik istek sayısını bir zaman aralığı içinde sınırlar.Bir azaltma eşiği aşılırsa, İş Ortağı Merkezi istemciden gelen diğer tüm istekleri bir süre için sınırlar.

- Azaltma minimum kapalı kalma süresi sağlar.

- Yüksek hacimli istekleri azaltarak tüm iş ortakları için tutarlı bir performans elde edilebilir.

### <a name="summary"></a>Özet

### <a name="categories"></a>Kategoriler

- Tarih: 2021-07-09
- Hazır Olma

### <a name="impacted-audience"></a>Etkilene hedef kitle

Bulut Çözümü Sağlayıcısı (CSP) doğrudan fatura iş ortakları ve mevcut müşterilerin adres ayrıntılarını yeni veya güncelleştiren dolaylı sağlayıcılar

### <a name="details"></a>Ayrıntılar

Microsoft güven üzerinde çalışır. CSP programında müşteri aboneliklerinin işlemesi için uyumlu, güvenli ve güvenli bir müşteri adresi doğrulama yöntemi sağlama konusunda kararlıyız. 13 Temmuz 2021'den itibaren Adresi Doğrula API'sini değişiklikler canlı yayında olacak.
 
Bu değişikliklerin yalnızca Adresi Doğrula API'sini etkilediğini unutmayın. Müşteri Oluşturma ve Faturalama Profili Güncelleştirme API'leri bundan etkilenmez. Şu anda önerilen adresin Müşteri Oluşturma API'si ile kullanılmasına gerek yok ancak bu kesinlikle önerilir.

Yanıt aşağıdaki durum iletilerinden birini geri dönecektir:

| Durum | Açıklama | Döndürülen önerilen adreslerin sayısı |
|------------|---------------|------------------|
| Doğrulanan sevk özellikli  | Adres doğrulanır ve sevk edilebilir.  | Tek  |
| Doğrulanamayan  | Adres doğrulandı.  | Tek  |
| Etkileşim gerekli  | Önerilen adresler/(es) önemli ölçüde değiştirilmiş ve Kullanıcı onayına ihtiyaç duyuyor.  | Tek  |
| Cadde kısmi  | Adreste verilen cadde kısmi ve daha fazla bilgi gerekiyor.  | Birden çok — en fazla 3  |
| Şirket içi kısmi  | Verilen şirket içi (bina numarası, paket numarası ve diğerleri) kısmi ve daha fazla bilgi gerekiyor.  | Birden çok — en fazla 3  |
| Birden çok  | Adreste kısmi olan birden çok alan vardır (büyük olasılıkla cadde kısmi ve şirket Içi kısmı da dahil).  | Birden çok — en fazla 3  |
| Hiçbiri  | Adres yanlış.  | Hiçbiri  |
| Doğrulanmamış  | Adres, doğrulama işlemi aracılığıyla gönderilemedi.   | Hiçbiri  |

Adres doğrulama API 'SI aracılığıyla doğrulanacak bir adres gönderildikten sonra, aşağıdaki yanıt şeması döndürülür:

```csharp

// <summary>
/// Object represents the address validation response.
/// </summary>

public class AddressValidationResponse
{
   /// <summary>
   /// Gets or sets the original address
   /// </summary>
   /// <value>
   /// Original Address
   /// </value>
   public Address OriginalAddress { get; set; }

   /// <summary>
   /// Gets or sets the suggested addresses
   /// </summary>
   /// <value>
   /// Suggested Addresses
   /// </value>
   public List<Address> SuggestedAddresses { get; set; }

   /// <summary>
   /// Gets or sets the validation status
   /// </summary>
   /// <value>
   /// Status
   /// </value>
   public string Status { get; set; }

   /// <summary>
   /// Gets or sets the validation message
   /// </summary>
   /// <value>
   /// Validation Message
   /// </value>
   public string ValidationMessage { get; set; }

```

Birleşik Devletler için, yalnızca posta kodu için beş basamak girerseniz, yanıt posta kodu satırı için dört basamaklı ek bir sonek döndürdüğüne unutmayın.

```C#
}
```

Bu güncelleştirmeler hızla işlem yapmanıza yardımcı olur ve çalışanlarınızın sertifikalarını yenilemesini, hem kimlik bilgilerini hem de MPN program durumunuzu etkin halde tutmayı hatırlatır.

- Bir test merkezine gitmek ve Sertifika sınavlarını yeniden almak için ödeme yapmak yerine, çalışanlarınız Microsoft Learn ziyaret eder ve en son teknoloji güncelleştirmelerinde bunları sınayan ücretsiz bir sertifika yenileme değerlendirmesi iletir.

- Değerlendirmenin iletilmesi, sertifika süre sonunu bir yıl boyunca uzatır.
- Bu işlem her yıl yinelenir ve yeteneklerin güncel olmasını sağlar.

Bu yeni yıllık yenileme süreci ile hizalamak için, Microsoft ayrıca iki yıldan bir yıla kadar rol tabanlı ve özel sertifikalar için geçerlilik zaman çerçevesini değiştiriyor. 30 Haziran 2021 ' den itibaren, kazanılan tüm rol tabanlı ve özel sertifikalar bir yıl boyunca geçerli olacaktır.
