## <a name="new-commerce-experience-in-csp-for-azure-expanding-to-government-community-cloud-gcc"></a><a name="16"></a>Azure için CSP 'de yeni ticaret deneyimi Government Community Cloud genişletiliyor (GCC)

### <a name="summary"></a>Özet

24 ağustos 2021 ' de, Azure için Bulut Çözümü Sağlayıcısı (CSP) programındaki yeni ticaret deneyimi tüm Government Community Cloud (GCC) iş ortakları tarafından kullanılabilir.

### <a name="categories"></a>Kategoriler

- Tarih: 2021-07-26
- Özellikler

### <a name="impacted-audience"></a>Etkilenen hedef kitle

GCC iş ortakları CSP programı aracılığıyla deneyimidir

### <a name="details"></a>Ayrıntılar

24 ağustos 2021 ' den itibaren, Azure için CSP 'deki yeni ticari deneyimin tüm GCC iş ortaklarına kullanılabilirliğini duyurmaktan mutluluk duyuyoruz. Bu deneyim, müşterilerin Azure hizmetlerini satın alıp tüketme yöntemini kolaylaştırır ve geliştirir. Ayrıca, CSP programındaki iş ortaklarına satış momalları genelinde Azure fiyatlandırmasının tutarlı bir görünümü, genel tutarlılık için ABD Doları fiyatlandırması, faturalama tarihi hizalaması ve Azure maliyet yönetimine erişim hakkı verir. Ayrıca, [Iş Ortağı Merkezi 'NDE](../azure-plan-transition.md) CSP iş ortaklarının müşterileri Azure planı KAPSAMıNDAKI mevcut CSP Azure teklifleriyle Azure hizmetlerine taşımasına yardımcı olan bir işlevselliğe de sahipsiniz.

### <a name="next-steps"></a>Sonraki adımlar

Yeni Azure ticari deneyimini tanıtan ve ek bilgiler sağlayan birkaç kaynak mevcuttur. [CSP program güncelleştirmeleri kaynak galerisinde](https://partner.microsoft.com/resources/collection/new-azure-experience-in-csp#/)en son SSS, Decks, videolar ve daha fazlasını bulun.

## <a name="enhancement-of-tool-for-transfer-of-azure-subscriptions-between-partners-in-the-cloud-solution-provider-csp-program"></a><a name="15"></a>Bulut Çözümü Sağlayıcısı (CSP) programındaki iş ortakları arasında Azure aboneliklerinin aktarılması için araç geliştirme

### <a name="summary"></a>Özet

Mevcut Azure aboneliği geçiş aracı, bir müşterinin önceki Azure teklifinin CSP aboneliklerinde iş ortakları arasında aktarım yapabilmesi için geliştirilecektir.

### <a name="categories"></a>Kategoriler

- Tarih: 2021-07-21
- Özellikler

### <a name="impacted-audience"></a>Etkilenen hedef kitle

CSP doğrudan fatura ortakları ve dolaylı sağlayıcılar ve dolaylı sağlayıcılar aracılığıyla satıcılar

### <a name="details"></a>Ayrıntılar

21 Temmuz 2021 ' den itibaren, [mevcut Azure aboneliği geçiş aracı](./2020-september.md#15) , bu abonelikleri yeni Azure teklif planına geçirirken, bir müşterinin önceki Azure teklifinin CSP aboneliklerinde iş ortakları arasında aktarabilmesi için geliştirilecektir. İş ortaklarının yapması gereken araç ve işlem, ortağın iş ortağı yeni Azure teklif planı aktarımları ile aynı kalacaktır. [İş ortağı kılavuzunda](https://partner.microsoft.com/resources/detail/step-by-step-guide-transfer-azure-subscriptions-between-csp-partner-new-commerce-experience-pdf)ayrıntılı bilgileri bulun.

Azure aboneliği geçiş araçları, CSP iş ihtiyaçlarını müşterilerin gelişen iş ihtiyaçlarına uygun yollarla Azure 'un yönetmesine olanak sağlayan CSP programındaki Azure için yeni ticaret deneyimini genişletmektedir. İş ortakları, Azure geçişleri için bu kullanımı kolay self servis aracı ile zamandan tasarruf edebilir. Faturalandırma sahipliğinin geçişi sırasında Azure hizmetlerinde bir kesinti olmayacaktır.

#### <a name="key-capabilities-and-considerations"></a>Temel yetenekler ve önemli noktalar

- Önceki Azure teklifinden yeni Azure teklif planına bir aboneliği geçmek, faturalandırma para birimini değiştirebilir. Yeni Azure teklifinde iş ortakları, [Avrupa Birliği/Avrupa Serbest Ticaret Birliği ve Birleşik Krallık bölgeleri](./2021-june.md#9)dışında müşterilerin konum para birimi ' nde faturalandırılır. bu bölgeler, Ağustos ayında, iş ortaklarının konum para birimi ' nde faturalandırılır.

- Azure ayırmaları aktarılmaz.

- Geçiş, müşteri kararına göre çalıştırılır ve yalnızca her iki iş ortağı açık eylemde bulunduğunda gerçekleşir.

- Geçiş, müşteri kesinti süresi veya Azure hizmetlerine etki olmadan anında gerçekleşir.

- Iş Ortağı Merkezi 'nde Bu araç deneyimi, doğrudan fatura ortakları ve dolaylı sağlayıcılar için kullanılabilir. Satıcıların bu aracı adına kullanması için dolaylı sağlayıcılarla çalışması gerekir.

### <a name="next-steps"></a>Sonraki adımlar

- [Bu iş ortağı Galerisi sayfasında](https://partner.microsoft.com/resources/collection/transition-tool-azure-subscriptions-new-commerce-experience-csp#/)güncelleştirilmiş kaynakları gözden geçirin.

## <a name="changes-to-azure-offer-availability-related-to-the-new-commerce-experience-in-csp-for-azure"></a><a name="14"></a>Azure için CSP 'deki yeni ticari deneyim ile ilgili Azure teklif kullanılabilirliğine yönelik değişiklikler

### <a name="summary"></a>Özet

bu iletişim, yeni ticaret deneyimi için Bulut Çözümü Sağlayıcısı (CSP) ' de Azure teklif kullanılabilirliği 'ndeki değişikliklere yönelik bir güncelleştirmedir.

### <a name="categories"></a>Kategoriler

- Tarih: 2021-07-21
- Özellikler

### <a name="impacted-audience"></a>Etkilenen hedef kitle

CSP programı aracılığıyla iş ortakları deneyimidir

### <a name="details"></a>Ayrıntılar

CSP programındaki önceki Azure teklifi kullanılabilirliğine yapılan değişiklikler.

Ocak ayında, CSP programındaki önceki Azure teklifi kullanılabilirliğine yaklaşan değişiklikler hakkında iş ortakları bilgilendirildi. Ocak bildiri ayrıntıları için Ayrıca bkz. [ocak 2021 duyuruları](./2021-january.md#12) .

Önceki Azure teklifinde değişiklik gösteren üç aşama olacağını duyurduk ve bu üç aşamadan her birinin başlangıç tarihinden önce iş ortaklarına altı ay önce haber göndereceğiz.

Ocak ayında duyurulduğu gibi, 1. aşama için başlangıç tarihi bugün 21 Temmuz 2021 ' dir. Ayrıca, 2. aşama başlangıç tarihinin 1 Şubat 2022 olacağını duyuruyoruz.

#### <a name="three-phases"></a>Üç aşama

**1. Aşama: 21 Temmuz 2021 ' den başlayarak**

1. aşamada, tüm yeni Azure CSP programı işleri yeni Azure teklifi (Azure planı) üzerinde olacaktır. Aşağıdaki yönergeler geçerlidir:

- CSP programındaki iş ortakları ve müşteriler arasındaki tüm mevcut satıcı ilişkileri için, müşteri zaten önceki Azure teklifini satın almışsa iş ortağı Transact 'ye devam edebilir ve müşteri önceki Azure teklifini bu iş ortağından satın almaya devam edebilir.

- Önceki Azure teklifi, CSP programındaki yeni iş ortakları tarafından artık deneyimidir kullanılamayacak.

- Önceki Azure teklifi artık CSP programındaki bir iş ortağı aracılığıyla satın alan yeni müşteriler tarafından kullanılamaz.

- Tüm yeni satıcı ilişkileri veya yeni müşteri alımları için iş ortakları yalnızca yeni Azure teklifini (Azure planı) kullanabilir.

- İş ortakları ve müşteriler arasındaki mevcut tüm satıcı ilişkileri için, müşteri hiçbir daha önce Azure 'u hiç satın almadıysanız iş ortağı yalnızca Transact ve müşteri yalnızca yeni Azure teklifini satın alabilir.

**2. Aşama: 1 Şubat 2022 ' den başlayarak**

2. aşama 'da, teşvikleri ve iş ortağı kenar boşluğu fırsatı önceki Azure teklifinden kaldırılır.

CSP 'de önceki Azure teklifi

- CSP 'deki önceki Azure teklifiyle ilişkili iş ortağı marjı fırsatı ve teşvikleri kaldırılır.

- CSP 'deki önceki Azure teklifinin fiyatı, Azure için tahmini perakende fiyata eşit olacaktır.  

CSP 'de yeni Azure teklifi (Azure planı)

- İş ortakları, CSP 'deki yeni Azure teklifi (Azure planı) üzerinde iş ortağı kazanılmış kredisi (PEC) kazanalmaya uygun olmaya devam edecektir.  

- Microsoft Commerce teşvikleri 'e kayıtlı iş ortakları, CSP 'deki yeni Azure teklifi (Azure planı) ile ilgili teşvikleri için uygun olmaya devam edecektir.  

- Gümüş bulut platformu uzmanlarından oluşan iş ortakları, CSP 'deki yeni Azure teklifi (Azure planı) ile ilgili ek teşvikleri için uygun olacaktır (2021 Ekim geçerlidir).

**3. Aşama: duyurulacak Tarih**

3. aşamada, önceki Azure teklifinde kalan tüm müşteriler, CSP programında (Azure planı) yeni Azure teklifine geçirilir. Önceki aşamalarda olduğu gibi, 1. aşama başlangıç tarihini altı ayda bir ön iş ortaklarına ileteceğiz.

#### <a name="partner-earned-credit"></a>İş ortağı kazanılmış kredisi

Yeni ticaret deneyimi sayesinde iş ortakları, PEC modeli aracılığıyla sürekli değer eklenmiş hizmetler için sürekli olarak tanınır. İş ortakları, Sürdürülebilir karlılık üreten, katma değerli hizmetler aracılığıyla müşteri başarısını iletmenize odaklanmaktadır.

## <a name="introducing-windows-365-hybrid-windows-for-a-hybrid-world"></a><a name="13"></a>Windows 365 ile tanışın: karma dünya için karma Windows

### <a name="summary"></a>Özet

Microsoft, Windows [365](https://aka.ms/M365EndpointPartner)ile Windows getiriyor.

### <a name="categories"></a>Kategoriler

- Tarih: 2021-07-20
- Modern Çalışma Alanı

### <a name="impacted-audience"></a>Etkilene hedef kitle

Bulut Çözümü Sağlayıcısı (CSP) programı aracılığıyla işlem yapılan iş ortakları

### <a name="details"></a>Ayrıntılar

14 Temmuz'da Microsoft, Windows [365](https://aka.ms/M365EndpointPartner)ile buluta Windows duyurdu. Windows 365, Windows 10 veya Windows 11'de deneyim yaşamanın, kişiselleştirilmiş uygulamalarınızı, ayarlarınızı ve içeriğinizi Microsoft bulutundan herhangi bir cihaza akışa alamanıza yönelik yeni bir yol sunar.

Windows 365 Cloud PC'nize hoş geldiniz:

- **Cihazlar arasında akış**

    Bulut bilgisayarınıza anında önyükler ve cihaz Windows akışı yapabilirsiniz.  
- **Her zaman hazır ve güncelleştirilmiş**

    Buluttan akışı yapılan ve birlikte seyahat eden Windows 365, nerede olursanız olun, çalışmak istediğiniz cihazda her zaman hazırdır ve her zaman güncelleştirilir. Bu, Microsoft bulutu tarafından Windows ve çok yönlü bir deneyimdir.
- **Kişisel ve tanıdık**

    Windows 365 kişiselleştirilmiş ve tüm ayarlarınız ve içeriğiniz hakkında bilgi sahibidir. Bu nedenle, Bulut bilgisayarınıza geri dönüp onu bıraktığınız yerdedir.
- **Ölçeklenebilir ve esnek**

    Windows 365, bulutun gücü ve bulut bilişim ilkelerini kullanarak tasarım Sıfır Güven. Bilgiler güvenli hale getirildi ve cihazda değil bulutta depolanır.
- **Tahmin edilebilir maliyetler**

    Windows 365, Bulut bilgisayarları satın almayı kullanıcı başına aylık fiyatlandırmayla tahmin edilebilir hale gelir.

- **Satın alma, dağıtma ve yönetme basit**

    Windows 365, diğer çözümlerden farklı olarak arkalarında sanallaştırma kullandığı için tedarik, dağıtım, yönetim ve güncelleştirmeleri basitleştiriyor.

Windows 365, müşterilerin 2 Ağustos 2021'den [başlayarak windows365.com, Kurumsal Anlaşma](https://windows365.com)veya Bulut Çözümü Sağlayıcısı satın almaları için kullanılabilir.

### <a name="next-steps"></a>Sonraki adımlar

- blog gönderimiz aracılığıyla Windows 365 hakkında [daha fazla bilgi edinebilirsiniz.](https://www.microsoft.com/microsoft-365/blog/?p=237041?)

### <a name="details"></a>Ayrıntılar

Microsoft güven üzerinde çalışır. CSP programında müşteri aboneliklerinin işlemesi için uyumlu, güvenli ve güvenli bir müşteri adresi doğrulama yöntemi sağlama konusunda kararlıyız. 13 Temmuz 2021'den itibaren Adresi Doğrula API'sini değişiklikler canlı yayında olacak.
 
Bu değişikliklerin yalnızca Adresi Doğrula API'sini etkilediğini unutmayın. Müşteri Oluşturma ve Faturalama Profili Güncelleştirme API'leri bundan etkilenmez. Şu anda önerilen adresin Müşteri Oluşturma API'si ile kullanılmasına gerek yok ancak bu kesinlikle önerilir.

Yanıt aşağıdaki durum iletilerinden birini geri dönecektir:

| Durum | Açıklama | Döndürülen önerilen adres sayısı |
|------------|---------------|------------------|
| Doğrulanabilir  | Adres doğrulanır ve adresine gönderebilirsiniz.  | Tek  |
| Doğrulandı  | Adres doğrulandı.  | Tek  |
| Etkileşim gerekiyor  | Önerilen adresler önemli ölçüde değiştirildi ve kullanıcı onayı gerekiyor.  | Tek  |
| Sokak kısmii  | Adreste verilen sokak kısmidir ve daha fazla bilgiye ihtiyaç vardır.  | Birden çok — en fazla 3  |
| Şirket içi kısmi  | Verilen şirket içi (bina numarası, paket numarası ve diğerleri) kısmi ve daha fazla bilgi gerekiyor.  | Birden çok — en fazla 3  |
| Birden çok  | Adreste kısmi olan birden çok alan vardır (büyük olasılıkla cadde kısmi ve şirket Içi kısmı da dahil).  | Birden çok — en fazla 3  |
| Hiçbiri  | Adres yanlış.  | Hiçbiri  |
| Doğrulanmamış  | Adres, doğrulama işlemi aracılığıyla gönderilemedi.   | Hiçbiri  |

Adres doğrulama API 'SI aracılığıyla doğrulanacak bir adres gönderildikten sonra, aşağıdaki yanıt şeması döndürülür:

```csharp

// <summary>
/// Object represents the address validation response.
/// </summary>

public class AddressValidationResponse
{
   /// <summary>
   /// Gets or sets the original address
   /// </summary>
   /// <value>
   /// Original Address
   /// </value>
   public Address OriginalAddress { get; set; }

   /// <summary>
   /// Gets or sets the suggested addresses
   /// </summary>
   /// <value>
   /// Suggested Addresses
   /// </value>
   public List<Address> SuggestedAddresses { get; set; }

   /// <summary>
   /// Gets or sets the validation status
   /// </summary>
   /// <value>
   /// Status
   /// </value>
   public string Status { get; set; }

   /// <summary>
   /// Gets or sets the validation message
   /// </summary>
   /// <value>
   /// Validation Message
   /// </value>
   public string ValidationMessage { get; set; }

```

Birleşik Devletler için, yalnızca posta kodu için beş basamak girerseniz, yanıt posta kodu satırı için dört basamaklı ek bir sonek döndürdüğüne unutmayın.

```C#
"addressLine1": "1 Micro Way", 
"postalCode": "98052" 
},

}
```

Bu güncelleştirmeler hızla işlem yapmanıza yardımcı olur ve çalışanlarınızın sertifikalarını yenilemesini, hem kimlik bilgilerini hem de MPN program durumunuzu etkin halde tutmayı hatırlatır.

Yeni sertifika yenileme işlemi nasıl çalışır?  

- Bir test merkezine gitmek ve Sertifika sınavlarını yeniden almak için ödeme yapmak yerine, çalışanlarınız Microsoft Learn ziyaret eder ve en son teknoloji güncelleştirmelerinde bunları sınayan ücretsiz bir sertifika yenileme değerlendirmesi iletir.
- Sertifika sona erme tarihleri yakınında, Microsoft 'tan e-posta bildirimleri alırlar ve bunları Microsoft Learn sertifika yenileme değerlendirmesine yönlendirirler.
- Çevrimiçi öğrenme yolları, değerlendirme için hazırlanmaya yönelik çalışma malzemesini içerir ve bu sayede ihtiyaç duydukları kadar çok zaman deneyebilirler.
- Değerlendirmenin iletilmesi, sertifika süre sonunu bir yıl boyunca uzatır.
- Bu işlem her yıl yinelenir ve yeteneklerin güncel olmasını sağlar.

Bu yeni yıllık yenileme süreci ile hizalamak için, Microsoft ayrıca iki yıldan bir yıla kadar rol tabanlı ve özel sertifikalar için geçerlilik zaman çerçevesini değiştiriyor. 30 Haziran 2021 ' den itibaren, kazanılan tüm rol tabanlı ve özel sertifikalar bir yıl boyunca geçerli olacaktır.

Microsoft temel bilgileri ve eski sertifika sertifikalarının muaf olduğunu ve yeni işlemden etkilenmediğini unutmayın.
